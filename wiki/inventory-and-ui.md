# Inventory and UI

Back: [Home](../index.md)

## 1) What this system is

This page covers the new beta inventory and the shared menu navigation flow.

## 2) Who should use it

- Anyone learning the new UI
- Players trying to manage gear/jewelry more cleanly
- Returning players who still expect the older command-only flow

## 3) Core rules (exact mechanics)

Main shortcut:

- `O` opens the beta inventory
- `/i` also opens it if command access is enabled

The beta inventory includes:

- main inventory
- hotbar
- armor slots
- amulet slot
- two ring slots
- live stat panel
- class panel
- talents panel
- abilities panel

Shared top tabs now link:

- Inventory
- Skills
- Quests
- Party
- Dungeon Finder

This top navigation is now present across the major custom menus so you can move between them without bouncing through separate commands constantly.

## 4) Inventory behavior

Current supported behavior includes:

- drag and drop
- double-click equipping
- shift-click movement
- equipping/unequipping armor and jewelry in the same UI
- live stat updates as gear changes
- class, talents, and abilities from the same shell
- talent reset support from the talents tab

Important note:

- This inventory is still beta, so if you hit a weird UI bug, report it.

## 5) Why it matters

- Jewelry is now much easier to manage
- Class/talent/ability info is easier to reference while gearing
- Players can swap between progression menus much faster
- The UI now behaves more like a full MMO menu shell instead of isolated windows

## 6) Common mistakes

- Forgetting that `O` is now the main inventory shortcut
- Not using the shared top tabs
- Assuming ring/amulet slots are cosmetic only
- Ignoring the live stat panel while testing gear swaps
- Forgetting that the beta inventory now handles most normal inventory interaction patterns already
